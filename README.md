# Proyecto Avance 1: Arquitectura de Datos Fuente   
**Nombre**: Edith Chuico  
**Curso**: 30765  
## Resumen 
En el presente repositorio se obtiene la estructuración de un modleo Dual para una tienda de dispositivos electrónicos, en donde se indica la parte de inventario, que se realizará bajo una estructura no SQL. Y la sección de ventas, que se mantiene en una estructura SQL. 
Además de ello, se realiza un análisis exhaustivo sobre los problemas que pueden presentarse al momento de mezclar datos provenientes de dos tipos de bases de datos diferentes, donde la principal preocupación es la pérdida de integridad referencial entre sistemas. Se identifican y detallan problemas críticos de calidad de datos, como las discrepancias en precios históricos, la falta de estandarización en zonas horarias de las transacciones, y la potencial inconsistencia en la unicidad de identificadores, que son cruciales de resolver antes de construir el Data Mart de Business Intelligence.
